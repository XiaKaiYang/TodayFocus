# 2.2 四级串级 PID 控制架构设计说明

## 目标

将 `/Users/xiakaiyang/Desktop/quadrotor_presentation/full_presentation.html` 中 `2.2 四级串级PID控制架构` 这一页从“交互探索型框图”调整为“讲解引导型框图”，让听众在演讲过程中先抓住主链路，再理解 `z` 轴推力的旁路关系。

## 现状问题

当前页面主要依赖悬停显示信息，导致关键逻辑必须等交互后才完整可见。页面虽然展示了模块和信号，但讲解顺序不够显式，听众容易把“结构展示”和“输入输出细节”混在一起；同时底部说明中“工程优势”比“讲解主线”更突出，和后续 `2.3 各环路设计特性` 的分工不够清晰。

## 采用方案

采用“主链路 + 分支说明”的静态结构。页面中部以固定顺序展示 `位置环 → 速度环 → 姿态角环 → 角速度环 → 混控分配 → 电机执行` 的主讲解链路，并从速度环单独拉出 `z` 轴总推力支路进入混控分配，明确说明 `xy` 通道继续进入姿态内环，而高度通道直接参与推力合成。

## 页面结构

1. 标题下方增加一句导语，直接说明“先看主链路，再看 `z` 轴旁路”。
2. 中部主图改为静态编号框图，每个模块只保留一条核心“输出是什么”信息。
3. 速度环到混控分配的 `z` 轴支路单独着色，并加标签说明其不经过姿态角环。
4. 底部说明卡片从“工程优势”改为“讲解顺序”和“本页重点”，避免与 `2.3` 的详细表格重复。

## 内容边界

本页只负责回答“控制信号是怎样逐层传递的”。各环的详细输入、输出、带宽和功能差异继续留给 `2.3 各环路设计特性` 展开，这样章节逻辑变成“2.1 先总述，2.2 看信号流，2.3 再拆各层细节”。

## 实现说明

实现时删除当前 `2.2` 页面的动态生成脚本和悬停信息样式，替换为静态 HTML 结构和局部 CSS，减少讲解时的视觉噪声，并降低后续维护成本。
